# Contributing to Between Chapters

Between Chapters is a prompt framework, not a codebase. The most valuable contributions don't require any technical skill.

---

## What's most needed

### 1. Lived feedback
If you've run the 30 days — or even part of it — what happened? What questions landed? What felt off, too easy, too vague, or like it missed something important? Open an issue and describe it. Specific is better than general: "Day 12, question 2 felt repetitive after Day 5" is more useful than "Week 2 needs work."

### 2. Prompt improvements
The system prompt in `prompt.md` is the heart of the project. If you have suggestions for how to sharpen the weekly escalation, the four themes, or the Day 30 synthesis instructions, open a PR with your proposed changes and a brief explanation of what problem you're solving.

### 3. Translations
The prompt works in any language. If you've tested it in a language other than English and it holds up, a translated `prompt.[lang].md` in a `translations/` folder would extend the project's reach significantly.

### 4. Forks and adaptations
Built a version of this for a specific context — grief, career change only, couples, teams? Share it. Open an issue with a link or open a PR to add it under an `adaptations/` folder.

---

## What's not needed right now

- A web app (the single-thread approach is intentional)
- Platform-specific setup guides (the prompt is platform-agnostic by design)
- Feature requests that add complexity to the user experience

---

## How to open a PR

1. Fork the repo
2. Make your changes
3. Open a PR with a clear description of what you changed and why
4. If it's a prompt change, describe what the old version got wrong or missed

---

## Code of conduct

This project deals with people in vulnerable moments of their lives. Keep that in mind in all discussions. Be direct, be honest, be kind.
